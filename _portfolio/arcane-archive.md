**Type.** Personal project / undergraduate thesis project.

**Context.** ArcaneArchive was motivated by the difficulty of managing highly specialized subculture fashion collections using mainstream wardrobe or shopping applications. These applications often failed to capture domain-specific attributes such as silhouette, brand lineage, fabric texture, rarity, condition, styling context, and community-specific taxonomy.

**My contribution.** I treated the project as a small-scale ontology and database design problem. I independently designed the ontology, database schema, application architecture, and user interface. I defined item entities, attribute hierarchies, classification rules, and collection-management workflows.

I implemented the Android prototype using Kotlin, SQLite, and Android Studio. I also conducted small-scale informal testing with community members and refined the classification design based on feedback.

**Research relevance.** Although this is an earlier personal project, it reflects my interest in turning informal domain knowledge into structured data systems.
